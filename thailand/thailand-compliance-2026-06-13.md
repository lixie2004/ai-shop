> **报告日期**：2026-06-13
> **目标市场**：Thailand（泰国）

## 一、合规框架概览

### 1.1 泰国电商监管体系

| 监管机构 | 职责 | 影响 |
|------|------|------|
| **FDA（泰国食品药品监督管理局）** | 食品/药品/化妆品/保健品注册 | 美妆/健康品类必须 |
| **NBTC（泰国国家广播和通信委员会）** | 电信设备/无线产品认证 | 电子品类必须 |
| **TISI（泰国工业标准协会）** | 工业产品标准认证 | 部分品类需要 |
| **泰国海关** | 进口关税/检验检疫 | 所有进口商品 |
| **PDPA 委员会** | 数据保护 | 所有电商卖家 |
| **消费者保护委员会** | 消费者权益 | 所有卖家 |

## 三、进口法规

### 3.1 进口流程

```
1. 准备文件（发票、装箱单、原产地证）
2. 申请进口许可证（如需要）
3. 海关申报
4. 缴纳关税 + VAT
5. 检验检疫（如需要）
6. 放行
```

### 3.2 禁止/限制进口产品

| 类别 | 说明 |
|------|------|
| 禁止 | 假冒产品、违禁药品、色情内容 |
| 限制 | 食品、药品、化妆品、电子产品（需认证） |
| 免税 | 价值 ≤ THB 1,500 的进口商品 |

### 3.3 原产地规则

- **中国-东盟自贸协定**：部分产品可享受优惠关税
- **原产地证（Form E）**：可申请关税减免
- **建议**：办理 Form E 降低关税成本

## 四、PDPA 数据保护合规

### 4.1 泰国 PDPA 要点

| 要求 | 说明 | 影响 |
|------|------|------|
| 数据收集同意 | 必须获得用户明确同意 | 网站/APP 需有隐私政策 |
| 数据使用限制 | 只能用于声明目的 | 不能滥用用户数据 |
| 数据删除权 | 用户可要求删除数据 | 需建立删除流程 |
| 数据泄露通知 | 72小时内通知监管机构 | 需有应急预案 |
| 跨境数据传输 | 需满足特定条件 | 中国卖家需注意 |

### 4.2 合规建议

1. **隐私政策**：泰语版本，明确数据收集和使用
2. **Cookie 同意**：网站需有 Cookie 同意弹窗
3. **数据安全**：加密存储用户数据
4. **第三方共享**：明确告知用户数据共享情况
